# Emotion Output — Design Spec

**Date:** 2026-05-19  
**Worktree:** emotion-output  
**Scope:** Character roleplay conversations only (`Conversation → Message`)

---

## Overview

Each character message in a roleplay conversation will include a structured emotional state alongside the dialogue text. The emotion is generated by the LLM before writing the dialogue, fed back into subsequent prompts so other characters can react to it, and displayed in the UI.

---

## Consumers

| Consumer | What they use |
|---|---|
| Human reader | Colored dot (emotion) + size/ring (intensity) + tooltip (subtext) |
| Other characters | Full emotion object injected into conversation history |
| Evaluator | `emotion` + `intensity` tracked per turn for arc consistency scoring |

---

## Emotion Schema

**Enum (8 values):** `Joy | Trust | Fear | Surprise | Sadness | Disgust | Anger | Anticipation`

Plutchik's 8 primaries cover the full psychological spectrum. The LLM always picks one.

**Structure:**

```typescript
// packages/shared/schema/emotion.ts
export const EmotionSchema = z.object({
  emotion: z.enum([
    "Joy", "Trust", "Fear", "Surprise",
    "Sadness", "Disgust", "Anger", "Anticipation",
  ]),
  intensity: z.enum(["low", "medium", "high"]),
  subtext: z.string().max(120),
});
export type Emotion = z.infer<typeof EmotionSchema>;
```

## Data Model

`Message` gains three non-optional fields with defaults (no null checks anywhere):

```prisma
model Message {
  id             String   @id @default(uuid()) @db.Uuid
  conversationId String   @db.Uuid @map("conversation_id")
  characterId    String   @db.Uuid @map("character_id")
  content        String
  emotion        String   @default("Joy")
  intensity      String   @default("low")
  subtext        String   @default("")
  createdAt      DateTime @default(now()) @map("created_at")

  conversation Conversation @relation(fields: [conversationId], references: [id])
  character    Character    @relation(fields: [characterId], references: [id])
  @@index([conversationId])
  @@map("messages")
}
```

Migration: `ALTER TABLE messages ADD COLUMN emotion TEXT NOT NULL DEFAULT 'Joy'` etc.

`MessageRecordSchema` in `packages/shared/schema/` is updated to include the three fields (non-optional).

## LLM Output Format

**Method:** XML-style delimiters (Anthropic-recommended pattern). More stable than prose instructions across long conversations; preserves streaming for the dialogue block.

**Format the LLM must produce:**

```
<emotion>{"emotion":"Fear","intensity":"high","subtext":"Trying not to show weakness"}</emotion>
<dialogue>I don't think we should go in there.</dialogue>
```

**Instruction added to `character-roleplay.hbs`** (replaces the final output instruction):

```handlebars
## Output Format

Always respond in this exact structure — no exceptions, no preamble:

<emotion>{"emotion":"[EMOTION]","intensity":"[INTENSITY]","subtext":"[SUBTEXT]"}</emotion>
<dialogue>Your next line of dialogue or action</dialogue>

Where:
- [EMOTION]: one of Joy | Trust | Fear | Surprise | Sadness | Disgust | Anger | Anticipation
- [INTENSITY]: low | medium | high
- [SUBTEXT]: one sentence — what {{name}} truly feels beneath the surface, in their internal voice

Feel first, then speak. The emotion block grounds the dialogue.

Example:
<emotion>{"emotion":"Fear","intensity":"high","subtext":"Trying not to show weakness in front of the others"}</emotion>
<dialogue>I don't think we should go in there.</dialogue>
```

## Conversation History Format

When building the next character's prompt, each line in the history changes from:

```
Aria: "I don't think we should go in there..."
```

To:

```
[Aria — Fear: high | Trying not to show weakness] "I don't think we should go in there..."
```

If `subtext` is empty (historical messages), the line shows:

```
[Aria — Neutral] "..."
```

This is constructed in `frontend/lib/conversation/next.ts` when assembling the conversation context.

---

## Stream Parser

The parser in `frontend/lib/conversation/next.ts` implements a two-phase state machine:

```
BUFFERING_EMOTION  →  (on </emotion>)  →  STREAMING_DIALOGUE  →  (on </dialogue>)  →  DONE
```

- **BUFFERING_EMOTION:** Accumulates tokens silently. Does not stream to client.
- **Transition:** On `</emotion>`, extracts and JSON-parses the emotion block. Emits an `emotion` SSE event to the client. Switches state.
- **STREAMING_DIALOGUE:** Strips the `<dialogue>` opening tag, then streams tokens live to the client.
- **DONE:** Strips `</dialogue>`, stops.

**Fallback:** If `</emotion>` has not appeared after 200 buffered tokens, the parser switches to `STREAMING_DIALOGUE` and saves `Joy` as emotion (DB default). The stream never hangs.

**New SSE event emitted at transition:**

```
event: emotion
data: {"emotion":"Fear","intensity":"high","subtext":"Trying not to show weakness"}
```

Dialogue tokens follow immediately after using the existing `text_delta` chunk type.

**On completion, saved to DB:**

```typescript
await prisma.message.create({
  data: {
    conversationId,
    characterId,
    content: dialogue,           // clean text, no tags
    emotion: emotion.emotion,
    intensity: emotion.intensity,
    subtext: emotion.subtext,
  },
});
```

## UI

### Per-message emotion indicator

Each message bubble shows a colored dot aligned to the character name:

```
┌─────────────────────────────────────────┐
│  Aria                          ● Fear   │
│  "I don't think we should                │
│   go in there..."                        │
│                       [hover → tooltip] │
└─────────────────────────────────────────┘
```

- Dot color: mapped from `emotion-dots.tsx` (8 Plutchik emotions already implemented)
- Dot size/style by intensity:
  - `low` → small dot, 60% opacity
  - `medium` → normal dot
  - `high` → normal dot + pulsing ring
- Tooltip on hover: shows `subtext`
- The dot appears **before** dialogue starts streaming (emitted with the `emotion` SSE event)

### Cast state panel

`cast-state.tsx` shows the current emotional state of every character in the scene, updated after each message:

```
┌──────────────────────────────────┐
│ CAST                             │
│  Aria    ● Fear    high          │
│  Marco   ● Anger   medium        │
│  Lena    ● Neutral  —            │
└──────────────────────────────────┘
```

Data source: last `Message` per `characterId` in the conversation. The `coherence` field present in the current preview component is **removed** — it is not in scope.

---

## Out of Scope

- Emotion output for agent chat (`AgentSession → AgentTurn`)
- `coherence` field
- Emotion history charts or arc visualization
- Emotion influencing character selection / orchestrator logic
